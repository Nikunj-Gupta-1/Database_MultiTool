# ASM Tool V1.2 Roadmap

This document outlines the planned improvements for the V1.2 release, focusing on performance, stability, and UI enhancements.

## 1. High Priority (Performance & Core)
*   **Concurrency for Takeover Checks:** Replace batched `threading.Thread` in `check_takeover` with `concurrent.futures.ThreadPoolExecutor(max_workers=20)`. Maintain `dig` via subprocess for DNS resolution to avoid adding dependencies.
*   **Nuclei Template Auto-Update:** Run `nuclei -update-templates` in a background thread on startup *only* if the last update was > 24 hours ago. Track `nuclei_last_update` timestamp in `config.json` and display it in the Settings UI.
*   **URL Parameter Deduplication (gau/waybackurls):** Normalize URLs by stripping parameter values (e.g., `?id=1` -> `?id=`) to create signatures. Store the *first* fully intact URL encountered for each signature to reduce noise while maintaining testable endpoints.
*   **Backend Vulnerability Sorting:** Sort the `vulns` list by severity (`critical` > `high` > `medium` > `low` > `info` > `unknown`) on the backend before writing to `nuclei_results.json` and passing to the LLM.

## 2. UI / UX Features (Medium Priority)
*   **Markdown Report Export:** Add a lightweight "Download Report (.md)" button below the LLM output in the UI. Generate the file client-side via a Blob URL (no PDF generation required).
*   **Scan Cancellation:** Add a `/api/scan/cancel` endpoint and a `cancel_requested` flag checked between major scan phases. Expose a "Cancel Scan" button in the UI.
*   **Per-Phase Timings:** Track elapsed time for each major scanning phase and append the duration to the `activity.log` (e.g., `[+] Phase Port Scanning completed in 47s`).
*   **Network Map Node Search:** Add a search input above the canvas. Typing highlights matching nodes (e.g., yellow stroke) and dims non-matching nodes.
*   **"Ask Friday" Context Injection:** Add an action button next to each vulnerability in the list. Clicking it switches to the Friday tab and pre-fills the chat with: `"Explain [template-id] and how to remediate it on [host]"`.

## 3. Infrastructure & Polish (Lower Priority)
*   **Config Validation:** Add schema validation in `load_config()` to ensure types (e.g., host strings) are correct, falling back to defaults if corrupted.
*   **Tool Version Display:** Create an `/api/tools/versions` endpoint that executes `<tool> -version` and displays the cached results in the Settings panel next to the tool status indicators.
*   **SSE Log Streaming:** Replace the frontend `setInterval` 1.2s polling with an `EventSource` connected to a new `/api/stream` endpoint, using a `threading.Condition` in Python for zero-latency, push-based logs.
